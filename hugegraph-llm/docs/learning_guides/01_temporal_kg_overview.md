---

## 一句话定义

**时序知识图谱 Agent 记忆** = 用带时间戳的知识图谱存储 Agent 的长期记忆，支持"3 天前我知道什么"、"旧信息自动衰减"、"新证据推翻旧结论"等时序推理。

它不是传统 KG 的"快照",而是一段"会随时间演化的关系网络"。

---

## 核心痛点（为什么需要这个方向）

### 现有 Agent 记忆方案的三大缺陷

| 缺陷 | 表现 | 后果 |
|------|------|------|
| **无时间维度** | 用户的"我住在上海"和 3 个月后的"我搬到北京"被同等对待 | 产生"用户在两个城市"的矛盾 |
| **静态衰减** | 所有记忆权重线性下降,无差异 | 重要事实被淡忘,琐事长期留存 |
| **冲突掩盖** | 新事实覆盖旧事实,旧事实被默默丢弃 | 无法溯源,Agent 决策无解释 |

### 时序 KG 如何解决

```
传统向量记忆:   "我住在上海" (vec) ──┐
                                    ├─> 两个相似向量并存,无法裁决
                "我搬到北京" (vec) ──┘

时序 KG 记忆:   "我住在上海"        (valid_from: 2026-01, valid_until: 2026-04)
                "我搬到北京"        (valid_from: 2026-04, supersedes: 上海)
                                       ↑ 自动建立"取代"边,可追溯
```

## 为什么这个方向现在火

1. **Mem0 / Zep Graphiti 验证可行**: 2025-2026 年 LLM Agent 框架统一接入时序记忆,行业共识形成
2. **长上下文 ≠ 长期记忆**: Claude/GPT-4 1M 上下文也解决不了"3 周前的对话细节",需要结构化记忆
3. **KDD 2026 / ICLR 2026 论文密集发布**: Memory³, Mem0, GraphRAG 等持续刷榜
4. **多模态时序数据爆发**: 用户行为/对话/IoT 全部带时间戳,需要统一存储

## 与 HugeGraph 核心差异化

| 维度 | Mem0 / Graphiti | HugeGraph 时序 KG |
|------|----------------|-------------------|
| **图规模** | 单机,万级实体 | **十亿级顶点/边,水平扩展** |
| **OLAP 多跳遍历** | 3-5 跳 | **Vermeer 引擎,60亿边 10+ 跳** |
| **时序推理** | Episode→Entity→Community 三层 | **可叠加 OLAP 统计/PageRank/社区发现** |
| **国产生态** | 绑定 OpenAI/Anthropic | **MiMo/通义/文心均可接入** |
| **开源协议** | Mem0 商业, Graphiti Apache-2.0 | **Apache-2.0** |
| **冲突检测** | 启发式 | **基于 Gremlin 子图遍历的精确检测** |
| **多租户** | 弱 | **原生多图隔离 (6+ 图并存)** |

## 这个方向的硬骨头 (技术挑战)

1. **时间索引设计**: 连续时间 vs 离散时间戳
2. **衰减函数选择**: 指数 / 幂律 / 阶梯式 (我们用指数 λ=0.05)
3. **冲突类型分类**: mutually_exclusive / temporal / granularity (KDD 2026 三类)
4. **证据链溯源**: 冲突裁决要可解释,不能黑盒
5. **大规模图性能**: 亿级事实下的实时检索 (<100ms)
