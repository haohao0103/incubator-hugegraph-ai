### Bugs discovered & fixed
- `InfoExtract._filter_long_id()` accesses `graph["vertices"]` directly (KeyError when empty schema)
  - Fix: Pre-initialize `context["vertices"]` and `context["edges"]`
- MiMo model name: must use `mimo-v2.5-pro`, NOT `gpt-4o-mini` (400 error)
- InfoExtract.run() requires BOTH `"chunks"` AND `"schema"` keys in context
- When schema is `{}` (falsy): uses text-based triple mode; when dict with keys: schema-based vertex extraction
- HugeGraph Server not running on localhost:8080 → graceful in-memory fallback

### Key insight: MiMo LLM behavior
- Text-based mode (empty schema): returns clean `(S,P,O)` triples — regex parseable ✅
- Schema-based mode (non-empty schema): returns Markdown tables — NOT regex parseable ❌
- Recommendation: use text-based mode for InfoExtract, or post-process markdown responses
